Semantic Control Plane

Source of truth for meaning

Typed, versioned, governed registries. Every semantic artifact has id · version · status · owners · changed_at. Graph-backed for sub-ms lookups.

🧬 Ontology

Concepts, relations, hierarchies, mereology.

Concept(kind: entity | metric | dimension | attribute | relation | event)

📐 Metric Registry

Formula as AST, unit, grain, additivity, SLA.

Metric.formula: Formula { expression, sql_template, verification_queries }

🧭 Dimension Registry

Hierarchies, attributes, default grain.

Dimension.hierarchy: [country → region → province → city]

🪪 Entity Registry

Canonical names, aliases, bindings to physical fields.

EntityBinding { entity_id, table_fqn, identifier_column, attr→col }

📚 Business Rule Registry

Typed & discriminated — no free text.

BusinessRule.category: metric_formula | filter_convention | …

🔗 Mapping Registry

MetricBinding: primary table, grain cols, join paths, filter predicates.

MetricBinding.dimension_join_paths: dim_id → JoinPath

🗣️ Vocabulary Registry

Per-domain lexicons: geo, metric terms, stopwords, months, locales.

config/vocab/{domain}.yaml

⏳ Time & Grain Registry

Fiscal calendar, historical/forecast boundary, indicator frequencies.

TimeSemantics.indicator_frequencies: { ind_id → daily|monthly|… }

🛡️ Policy Registry

SQL safety rules: detectors, severities, scopes, domain packs.

PolicyRule.detector: regex | ast_matcher | predicate_fn

❓ Ambiguity Registry

Known conflicts + disambiguation recipes + defaults.

AmbiguitySpec.candidate_resolutions + detection_hooks

🕸️ Ontology Store

In-memory LPG (networkx / rustworkx) + pgvector index; Postgres/Neo4j persistence. Optional SKOS/OWL export.

O(ms) queries: "which metrics use dim X?", "join path M → D"

🏛️ Governance Service

Owners, versions, approvals, deprecation, breaking-change detection.

status: draft | certified | deprecated · replacement_id · sunset_date
Storage & GitOps layout
  • /semantics/core/ — shared concepts (time, geography, money, person, org)
  • /semantics/domains/{qp,finance,…}/ — per-domain overlays (metrics, bindings, rules, vocab, time)
  • /semantics/_schemas/ — JSON Schemas per artifact type
  • PR merge → migrator syncs to Postgres with immutable version history
